Java Virtual Machine (JVM)

The JVM executes Java bytecode and enables Java’s “Write Once, Run Anywhere” principle.

  • Compilation to bytecode: .java → .class
  • Execution: Bytecode is interpreted / JIT-compiled to machine code.
  • Memory Management: Automatic heap management and garbage collection.

Example: From Source to JVM

// File: Demo.java public class Demo { public static void main(String[] args) { int x = 10; int y = 20; System.out.println("Sum = " + (x + y)); } } /* Steps: 1. Compile: javac Demo.java → Demo.class (bytecode) 2. Run: java Demo → JVM loads Demo.class and executes bytecode */

OOP Concepts

Object-Oriented Programming (OOP) is centered around objects and classes. Key pillars:

  • Encapsulation: Hiding data using private fields + public methods.
  • Inheritance: Reusing code by extending classes.
  • Polymorphism: Same method name, different implementations.
  • Abstraction: Hiding implementation details (abstract classes, interfaces).

Example: Simple OOP Model

class Animal { protected String name; public Animal(String name) { this.name = name; } public void speak() { // polymorphism (overridden in subclasses) System.out.println("Animal makes a sound"); } } class Dog extends Animal { // inheritance public Dog(String name) { super(name); } @Override public void speak() { // polymorphism System.out.println(name + " says: Woof!"); } } public class OopDemo { public static void main(String[] args) { Animal a = new Dog("Tommy"); // upcasting a.speak(); // runtime polymorphism } }

Exception Handling

Handle runtime errors gracefully so the program can recover or terminate cleanly.

  • try-catch-finally: Handle exceptions and cleanup.
  • throws: Declare that a method may throw exceptions.
  • Checked vs Unchecked: Checked must be handled; unchecked may not be.
  • Custom exceptions: Extend Exception or RuntimeException.

Example: try-catch-finally

public class ExceptionDemo { public static void main(String[] args) { try { int a = 10; int b = 0; int c = a / b; // will throw ArithmeticException System.out.println("Result = " + c); } catch (ArithmeticException e) { System.out.println("Cannot divide by zero: " + e.getMessage()); } finally { System.out.println("This will always execute."); } } }

Example: Custom Exception

class InvalidAgeException extends Exception { public InvalidAgeException(String message) { super(message); } } class Voter { public void checkAge(int age) throws InvalidAgeException { if (age < 18) { throw new InvalidAgeException("Age must be 18 or above"); } System.out.println("You can vote!"); } }

Collections Framework

The Collections Framework provides data structures and utilities for groups of objects.

  • List: Ordered, allows duplicates (ArrayList, LinkedList).
  • Set: No duplicates (HashSet, TreeSet).
  • Map: Key-value pairs (HashMap, TreeMap).
  • Queue/Deque: FIFO/LIFO structures (PriorityQueue, ArrayDeque).

Example: List and Map

import java.util.*; public class CollectionsDemo { public static void main(String[] args) { List<String> names = new ArrayList<>(); names.add("Umesh"); names.add("Rahul"); names.add("Priya"); for (String n : names) { System.out.println(n); } Map<Integer, String> studentMap = new HashMap<>(); studentMap.put(1, "Umesh"); studentMap.put(2, "Rahul"); System.out.println("Roll 1: " + studentMap.get(1)); } }

File I/O

Reading from and writing to files using Java I/O and NIO APIs.

  • java.io: Streams and readers/writers.
  • java.nio: Channels, buffers, Paths & Files.
  • Serialization: Persist objects using Serializable.

Example: Write and Read a Text File (java.io)

import java.io.*; public class FileDemo { public static void main(String[] args) throws IOException { String fileName = "data.txt"; // Write try (BufferedWriter writer = new BufferedWriter(new FileWriter(fileName))) { writer.write("Hello File I/O in Java"); } // Read try (BufferedReader reader = new BufferedReader(new FileReader(fileName))) { String line = reader.readLine(); System.out.println("File content: " + line); } } }

Generics

Generics enable type-safe classes and methods.

  • Generic classes: class Box<T>
  • Bounded type parameters: <T extends Number>
  • Wildcards: ? extends T, ? super T

Example: Generic Box

class Box<T> { private T value; public void set(T value) { this.value = value; } public T get() { return value; } } public class GenericsDemo { public static void main(String[] args) { Box<String> stringBox = new Box<>(); stringBox.set("Hello"); System.out.println(stringBox.get()); Box<Integer> intBox = new Box<>(); intBox.set(100); System.out.println(intBox.get()); } }

Streams & Lambda

Functional-style operations on collections, introduced in Java 8.

  • Lambda expressions: (args) -> expression
  • Streams: filter, map, sorted, collect.
  • Parallel streams: Use multiple cores easily.

Example: Lambda + Stream

import java.util.*; import java.util.stream.*; public class StreamDemo { public static void main(String[] args) { List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5, 6); List<Integer> evenSquares = numbers.stream() .filter(n -> n % 2 == 0) // keep even .map(n -> n * n) // square .collect(Collectors.toList()); System.out.println("Even squares: " + evenSquares); } }

JDBC

Java Database Connectivity (JDBC) connects Java code to relational databases.

  • DriverManager/DataSource: Create connections.
  • Statement/PreparedStatement: Execute SQL.
  • ResultSet: Read query results row by row.
  • Transactions: Use commit() and rollback().

Example: Basic JDBC Flow

import java.sql.*; public class JdbcDemo { public static void main(String[] args) { String url = "jdbc:mysql://localhost:3306/testdb"; String user = "root"; String password = "password"; String query = "SELECT id, name FROM students"; try (Connection con = DriverManager.getConnection(url, user, password); PreparedStatement ps = con.prepareStatement(query); ResultSet rs = ps.executeQuery()) { while (rs.next()) { int id = rs.getInt("id"); String name = rs.getString("name"); System.out.println(id + " - " + name); } } catch (SQLException e) { e.printStackTrace(); } } }
